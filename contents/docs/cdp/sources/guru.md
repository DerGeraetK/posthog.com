The Guru connector syncs your Guru knowledge base data into PostHog, including cards, collections, groups, and team members.

## Adding a data source

1. Go to the [Data pipeline page](https://app.posthog.com/data-management/sources) and select the **Sources** tab.
2. Click **+ New source** and select Guru by clicking the **Link** button.
3. Get your Guru API credentials:
   - **Username** - Your Guru account email address.
   - **API token** - A user API token generated by a Guru admin. Create one from [Settings > Apps and integrations > API access](https://app.getguru.com/settings/integrations/api-access) in Guru.
4. Enter your credentials in PostHog and click **Next**.
5. Select the tables you want to sync, set the sync method and frequency, then click **Import**.

<CalloutBox icon="IconWarning" title="Use a user token, not a collection token" type="caution">

Use a **user API token** for this integration. Collection tokens are read-only and scoped to a single collection, so they won't work for syncing all your Guru data.

</CalloutBox>

## Available tables

| Table | Description | Sync method |
| ----- | ----------- | ----------- |
| `cards` | Knowledge cards in your Guru workspace | Incremental |
| `collections` | Collections that organize cards | Full refresh |
| `groups` | User groups in your workspace | Full refresh |
| `members` | Team members in your workspace | Full refresh |

**Incremental** tables sync only new or updated records on each run. The `cards` table uses the `lastModified` field, so only cards changed since the last sync are fetched. **Full refresh** tables reload all data on each sync.
