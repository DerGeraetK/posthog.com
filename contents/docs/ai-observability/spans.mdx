Spans are units of work within an LLM [trace](/docs/ai-observability/traces). These are events that represent individual operations and discrete durations within your LLM application, like function calls, vector searches, or data retrieval steps, providing granular visibility into the execution flow.

## View span attributes

Expand any span row in the [trace list](https://app.posthog.com/ai-observability/traces) to view its attributes inline. Click the chevron on the left side of a row to reveal:

- **Attributes** – Custom key-value pairs you set on the span using OTel
- **Span details** – Intrinsic span properties like name, duration, and status

This lets you inspect span data without opening the full trace timeline.

## Span types

PostHog captures spans to track atomic operations that make up your LLM workflow. For example:

- **[Generations](/docs/ai-observability/generations)** - LLM calls and interactions
- **Vector database searches** - Document and embedding retrieval
- **Tool/function calls** - API calls, calculations, database queries  
- **RAG pipeline steps** - Retrieval, reranking, context building
- **Data processing** - Validation, chunking, formatting

For technical implementation details, see [manual capture](/docs/ai-observability/installation/manual-capture).
