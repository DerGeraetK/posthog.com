# Q3 2026

## Objective 1: Make HogQL-backed data exportable

### Motivation
* Enables a lot, if not all, products to be data sources of batch exports
  * Any dashboards, effectively replacing CSV exports (and offering ANY format exports)
  * Any warehouse views
  * And more...
* Have a single consistent way of exporting any type of data
  * If a new product adds new data sources using HogQL queries, they get batch exports “for free”
* Allow for more flexibility in the data models that can be exported

### What we’ll ship
* On-demand export of HogQL-backed data to supported destinations
* Scheduled sync of HogQL-backed data to supported destinations

## Objective 2: Agent-driven batch exports

### Motivation
* Improving experience for AI-pilled teams
* Make it easier for agents to create, validate and run batch exports

### What we’ll ship
* Skills for creating, validating, debugging, updating batch exports
* Integration UI for connecting to destinations

## Objective 3: The pipeline must grow

### Motivation
* Reduce time to deployment
* Ensure current customers are well served
* Faster, accurate, and self-driven debugging

### What we'll ship
* Work with devex on implementing CI improvements
* Better internal observability so we know what to improve
