### Q3 2026 Objectives

#### Feature flags <TeamMember name="Anders Asheim Hennum" photo />
Make it possible to run multiple experiments per feature flag. Rollout configurations will be more flexible and it will be easier to understand how feature flags and experiments are related. Implement the plan from the [RFC](https://github.com/PostHog/requests-for-comments-internal/blob/multiple-experiments-per-flag/engineering/2026-05-04-feature-flag-and-experiment-integration.md).

#### Dedicated exposure event <TeamMember name="Anders Asheim Hennum" photo />
Add an `experiment_exposure` event. This will:
1. Speed up queries by not relying on the `$feature_flag_called` event.
2. Make it clearer what exposure is and how it works, by no longer conflating a feature flag call with exposure.

##### Optimize precomputation and query performance <TeamMember name="Juraj Majerik" photo />

- Handle late-arriving events and person merges, which currently cause inaccuracies.
- Test the remaining query optimizations against real production queries.
- Give users better visibility into what happened to a query.
